# Release Copy

## 中文长版

我把最近一直在自己用的一套 OpenClaw 小红书内容工作流整理成开源了，名字叫 **Clawnote**。

它不是“自动发帖脚本”，而是一套更适合真实账号长期使用的 review-first workflow：

- 先做选题和来源验证
- 再写成小红书风格稿件
- 先发 Feishu 审稿
- 只有明确批准后，才做精确发布

这套东西最初是为了让我自己能稳定产出两类内容：

1. AI 新闻判断
2. OpenClaw 实践复盘

我后来发现，真正难的不是“让 AI 写一篇内容”，而是怎么把发布、审批、记忆、风险边界这些东西一起做对。

所以 Clawnote 重点解决的是：

- 如何把内容生产拆成 `research -> draft -> publish-assist`
- 如何把“先审后发”做成系统能力
- 如何把 daily memory 和 cron 接进内容运营流程
- 如何避免把高风险动作混成一个模糊的“自动化”

如果你也在折腾：

- OpenClaw
- AI 内容工作流
- 小红书/Rednote 运营
- 审批优先的发布链路

可以直接看看这个项目。

https://github.com/zhangifonly/clawnote
